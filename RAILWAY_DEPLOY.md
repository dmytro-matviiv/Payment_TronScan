# 🚀 Деплой на Railway

Інструкції для деплою Telegram бота на Railway.

## Крок 1: Підготовка проекту

Проект вже готовий до деплою! Всі необхідні файли створені:
- ✅ `Dockerfile` - для збірки контейнера
- ✅ `requirements.txt` - залежності Python
- ✅ `railway.json` - конфігурація Railway

## Крок 2: Створення проекту на Railway

1. Перейдіть на [Railway](https://railway.com/)
2. Увійдіть через GitHub (рекомендовано) або створить акаунт
3. Натисніть **"New Project"**
4. Виберіть **"Deploy from GitHub repo"**
5. Виберіть ваш репозиторій з ботом

## Крок 3: Налаштування змінних середовища

Після створення проекту, перейдіть в **Settings** → **Variables** та додайте:

```
TELEGRAM_BOT_TOKEN=8456055614:AAFeuIrPgQKDdfl_e9ULHi1oAJimxkaeLWM
TELEGRAM_CHANNEL_ID=@payment_trc20_001
TRONSCAN_API_TOKEN=3bfa787b-22a1-4c79-a2f5-b46dc062ee9f
TRON_ADDRESS=TYY4QoQ8nA4geSoYpueQ14DLFnHhYBKCiR
CHECK_INTERVAL=30
```

**Важливо:** Не додавайте `.env` файл в Git! Railway використовує змінні середовища зі своєї панелі.

## Крок 4: Деплой

1. Railway автоматично виявить `Dockerfile` та почне збірку
2. Після успішної збірки бот автоматично запуститься
3. Перевірте логи в розділі **Deployments** → **View Logs**

## Крок 5: Перевірка роботи

1. Перевірте логи - має бути повідомлення про успішний запуск
2. Перевірте Telegram канал - має прийти повідомлення про запуск бота
3. Надішліть тестову транзакцію та перевірте чи бот її відстежить

## Налаштування автодеплою

Railway автоматично деплоїть проект при кожному push в GitHub (якщо використовуєте GitHub).

## Моніторинг

- **Logs** - перегляд логів бота в реальному часі
- **Metrics** - моніторинг використання ресурсів
- **Settings** - налаштування проекту

## Вартість

Railway має безкоштовний план з обмеженнями. Для постійної роботи бота може знадобитися платний план (від $5/місяць).

## Troubleshooting

### Бот не запускається
- Перевірте логи на наявність помилок
- Переконайтеся, що всі змінні середовища встановлені правильно
- Перевірте правильність токенів

### Повідомлення не надходять в канал
- Перевірте чи бот додано в канал як адміністратор
- Перевірте правильність `TELEGRAM_CHANNEL_ID`
- Перевірте логи на помилки відправки

### Бот перезапускається
- Перевірте логи на помилки
- Можливо перевищено ліміти Railway (перевірте план)
- Перевірте налаштування `restartPolicyMaxRetries` в `railway.json`
